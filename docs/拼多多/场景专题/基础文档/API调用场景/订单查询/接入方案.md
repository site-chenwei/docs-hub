# 接入方案

更新时间：2021-02-22 15:45:46

### 

API介绍

-   [pdd.order.list.get 订单列表查询接口](https://open.pinduoduo.com/application/document/api?id=pdd.order.list.get)
    

根据成交时间查询订单

单次调用入参成交时间区间可达24小时，但是接口频率限流较大，建议在接入新用户时用于同步90天内历史订单，不建议同步实时订单使用。

-   [pdd.order.number.list.increment.get 订单增量接口](https://open.pinduoduo.com/application/document/api?id=pdd.order.number.list.increment.get)
    

根据订单更新时间获取增量订单，获取所有变动的订单

单次调用入参更新时间区间30分钟，接口承载能力强于列表接口，建议实时获取当前订单时使用

-   [pdd.order.information.get 订单详情接口](https://open.pinduoduo.com/application/document/api?id=pdd.order.information.get)
    

根据订单号获取单笔订单详情

已知订单号的情况下获取订单详情，可配合消息推送使用

### 

接入实施方案说明

订单数据获取主要分为初始化和增量获取两个步骤：

1.  初始化是用户新接入时通过订单列表接口将近90天内的订单获取到软件内
    
2.  增量获取则是实时获取店铺新增或发生更新的订单
    

下面的方案都会围绕着如何初始化和增量获取来进行说明

#### 

方案一

1.  新用户接入使用[pdd.order.list.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.list.get)同步历史订单（至多可获取90天内订单）。
    
2.  实时获取店铺新增或发生更新的订单请使用[pdd.order.number.list.increment.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.number.list.increment.get)接口，此接口不仅能够获取到新生成的订单，还可以获取到订单信息有任何更新的订单，此接口在使用时需要定时轮询以获取最新的订单详情。查询的页码可以通过返回的订单总数除以自己设置的page\_size获得。
    

注：

若增量同步有遗漏，可再调用[pdd.order.list.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.list.get)根据成交时间查询订单以做兜底补充。

知晓订单号的情况下可使用[pdd.order.information.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.information.get)订单详情接口获取订单信息。

#### 

方案二

1.  新用户接入使用[pdd.order.list.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.list.get)同步历史订单（至多可获取90天内订单）。
    
2.  通过消息推送服务客户端实时监听订单变更消息，再通过[pdd.order.information.get](https://open.pinduoduo.com/application/document/api?id=pdd.order.information.get)订单详情接口获取订单信息
    

注：消息推送服务相关介绍请查看[《消息推送使用指南》](https://open.pinduoduo.com/application/document/browse?idStr=D3B6DBF3A5CB57E5)
