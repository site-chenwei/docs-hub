# 多店铺关联

更新时间：2023-01-01 21:52:41

店铺关联oauth方案

服务商软件提供多店铺关联功能，便于用户在服务商系统账号体系下关联授权添加对多个店铺进行统一管理，但服务商自有关联操作不经过平台，存在店铺数据泄露风险，因此需将店铺关联能力收归开放平台。本改造方案目标是确保关联操作安全，被关联店铺信息不会泄露给第三方。

一、参与方

发起者——发起关联动作的操作者

被关联方——被关联店铺及其操作者

isv应用——服务商的软件系统

拼多多开放平台

二、流程

1.  发起者在isv应用上触发关联店铺操作，isv应用生成拼多多关联店铺oauth页面链接

2.  开平关联页面链接参数包含：isv处理关联的回调地址，ticket(发起者身份识别码，通过**pdd.pop.mall.bind.ticket.get**获取)

链接格式：[https://fuwu.pinduoduo.com/service-market/auth/relate?redirect\_uri=](https://fuwu.pinduoduo.com/service-market/auth/relate?redirect_uri=){client处理关联的回调地址}&ticket={发起者身份识别码}

\*发起者身份识别码：发起方触发关联操作时，isv应用通过接口向拼多多开平申请的短效身份识别码，包含isv应用信息（client\_id）和发起者身份信息（mall\_id），用于开平关联页面鉴定和展示发起者身份信息。回调isv应用时，isv也可以通过此识别码寻回发起者。

3.  被关联方通过链接进入拼多多关联页面并登录mms店铺账号

关联页面进行校验

校验项：回调地址、ticket、被关联方必须已购买且授权目标isv应用。

校验无误后，关联页面展示isv应用信息，发起方身份信息，以及提示操作风险

被关联方确认页面内容无误后，点击oauth页面“允许关联”按钮

![](https://t16img.yangkeduo.com/perse-open-api/2022-12-29/4166748c-81b4-4077-8408-35e1bd119985.png)

4.  拼多多开平产生一个关联凭证bindCode，并回调isv应用且传入ticket和bind\_code

5.  isv应用通过ticket定位发起者，并通过开放平台接口（pdd.pop.mall.bind.token.get）获取关联店铺名称和access\_token等信息，isv应用将发起者和被关联方完成关联操作。

注：步骤5仅返回被关联店铺已有授权的access\_token。

三、存量上报接口（pdd.pop.mall.bind.relation.report）

isv上报存量关联关系，入参：

clientId——isv应用id

externalUid——isv应用uid

inviterMallId——发起方店铺id

inviteeMallId——被关联方店铺id

bindType——关联方式

bindTime——关联时间
