# 物流发货

更新时间：2025-03-17 22:28:20

### 

发货基础说明

对于实物类商品，**已成交进入待发货**状态的订单（无售后或售后关闭且无审核）商家可以进行订单发货，生成运单号后，通过订单发货通知接口回传运单号和物流公司信息，接口调用成功订单进入“**已发货**”状态，消费者也能在我的订单中查看到物流轨迹信息。

### 

相关API

-   [pdd.logistics.online.send](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.online.send) 订单发货通知接口
    
-   [pdd.order.upload.extra.logistics](https://open.pinduoduo.com/application/document/api?id=pdd.order.upload.extra.logistics) 订单额外运单信息上传接口
    
-   [pdd.logistics.companies.get](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.companies.get) 快递公司查看接口
    
-   [pdd.logistics.ordertrace.get](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.ordertrace.get) 轨迹查询接口
    
-   [pdd.logistics.isv.trace.notify.subISV](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.isv.trace.notify.sub) 物流轨迹推送消息订阅接口
    

### 

操作流程

1、[订单查询](https://open.pinduoduo.com/application/document/browse?idStr=10388E8ACD79B689)获取所有待发货且无售后或售后关闭且无审核的订单；

-   订单状态：order\_status=1& refund\_status=1& risk\_control\_status=0
    
-   [审核订单说明](https://open.pinduoduo.com/application/document/browse?idStr=697D8F2EC5A48FEA)：平台对存在异常风险的订单判断为审核订单，针对审核订单会给予审核状态，审核字段状态字段：risk\_control\_status：值为 1 则为审核订单，值为 0 则为正常订单。
    

2、通过[pdd.logistics.companies.get](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.companies.get)获取快递公司编号

3、 若需要设置默认之外退货地址，通过售后接口[pdd.refund.address.list.get](https://open.pinduoduo.com/application/document/api?id=pdd.refund.address.list.get)获取商家退货地址id。

4、 通过[pdd.logistics.online.send](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.online.send)对订单进行发货。

-   该接口已支持修改发货信息： redelivery\_type：修改发货模式。枚举：1=首次发货：用于订单首次发货，仅待发货订单可传入； 2=修改发货：用于订单修改发货，调用成功后将会覆盖原发货信息，仅已发货订单可传入。 若不传入该字段，系统将默认为首次发货。
    

5、 若一笔订单存在多笔物流信息，则通过[pdd.order.upload.extra.logistics](https://open.pinduoduo.com/application/document/api?id=pdd.order.upload.extra.logistics)上传订单额外运单信息

6、通过[消息推送服务](https://open.pinduoduo.com/application/document/browse?idStr=D3B6DBF3A5CB57E5)客户端实时监听获取物流轨迹消息

-    [trace\_isv\_NotifyByMall](https://open.pinduoduo.com/application/document/message?secondCatId=5&id=20)：物流轨迹消息（按照商家维度）
    

使用场景满足以下任一条件：

1)     该运单是拼多多电子面单在该isv取号

2)     该运单调用拼多多发货接口给拼多多订单发货

**其他物流轨迹获取场景：**

1、[trace\_isv\_NotifyBySub](https://open.pinduoduo.com/application/document/message?secondCatId=5&id=23)物流轨迹消息(订阅推送)+通过[pdd.logistics.isv.trace.notify.sub](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.isv.trace.notify.sub)接口订阅快递单号的轨迹发生变更时，会发送此消息

使用场景：按物流单号维度订阅轨迹，isv可以对接[pdd.logistics.isv.trace.notify.sub](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.isv.trace.notify.sub)接口主动订阅物流运单号，来获取该运单的轨迹信息

2、[pdd.logistics.ordertrace.get](https://open.pinduoduo.com/application/document/api?id=pdd.logistics.ordertrace.get)轨迹查询接口

目前只有取号拼多多面单的isv有权限获取该运单号的物流轨迹

### 

常见问题FAQ

-   **问题: pdd.logistics.online.send订单发货通知接口，调用报错“接口超时，XX秒后重试”，怎么办？**
    

回答：建议订单使用拼多多电子面单发货，发货更加流畅打单更加迅速。使用拼多多电子面单的订单极速退款优化，降低恶意退款资损，各类活动优先审批，虚假发货流量免处罚等特权。

[拼多多电子面单对接文档](https://open.pinduoduo.com/application/document/browse?idStr=C4CD7938F68B0C7E)

-   **问题:pdd.logistics.online.send发货时候报“物流单号格式错误”**
    

回答：请使用pdd.logistics.companies.get快递公司查看接口获取正确的物流公司id入参到发货接口的logistics\_id；同一物流公司可能快递、快运、国际等会分多个id，请仔细辨别

-   **问题:pdd.logistics.ordertrace.get（轨迹查询接口）为什么获取不到轨迹？报错“该运单轨迹无权限”**
    

回答：轨迹查询接口只可查询通过同一应用（同client\_id）调用pdd.waybill.get接口获取面单的轨迹

-   **问题:pdd.logistics.online.send订单发货接口报错：商家使用的是顺丰快递发货的，选择的是 “特惠专配”的时效服务。 返回报文："加运费发顺丰订单，仅支持顺丰标快或顺丰特快发货"**
    

回答：[加运费发顺丰入口](https://mms.pinduoduo.com/orders/order/sfCarriage/goodList?spm=10193&refer_page_url=aHR0cHM6Ly9tbXMucGluZHVvZHVvLmNvbS9kYXh1ZS9kZXRhaWw/Y291cnNlSWQ9MjU3NA==)，加运费发顺丰入口-右上角有[《加运费发顺丰服务条款》](https://funimg.pddpic.com/msfe/2020-12-07/fd1a01c5-fce4-4ffc-b092-c4349ce8afff.pdf)

条款1.1说明，加运费发顺丰订单，需要使用顺丰快递，仅限顺丰标快、顺丰特快。

-   **问题:有一笔订单里商品数量较多，需要分个包裹发货，该如何上传运单号？**
    

回答：在将主运单号上传，订单发货后，调用额外运单信息接口上传订单其他运单号作为补充信息，可参考[额外运单信息改造白皮书](https://open.pinduoduo.com/application/document/doc?fullScreen=true&docReference=extra_logistics)。
