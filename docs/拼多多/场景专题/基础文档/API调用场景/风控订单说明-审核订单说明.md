# 审核订单说明

更新时间：2025-03-17 22:28:20

为了保障消费者和商家的权益，在订单同步服务中会展示订单审核状态。请开发者根据以下内容调整，务必在接入订单同步前完成改造。

### 

业务背景

平台对存在异常风险的订单判断为审核订单，针对审核订单会给予审核状态。

审核订单确认风险后会进行自动退款，确认无风险后会解除审核状态（大部分审核订单会解除审核，解除时间不定）。且不会对已发货订单标记审核。

为了让商家获知订单审核状态，且不对审核订单进行发货，订单信息将展示审核状

态，并隐藏收货人、收货电话、收货地址。

### 

字段说明

审核字段状态字段：risk\_control\_status：值为 1 则为审核订单，值为 0 则为正常订单。

当risk\_control\_status 为 1 时，“receiver\_address”、“receiver\_phone”、 receiver\_name”字段为空。

### 

处理建议

**用户端：**

1.  对于审核订单，开发者需要在订单列表与详情页的显著位置提示商家这是审核订单，并且开发者要隐藏收货信息，避免商家线下发货；
    
2.  对于审核订单，不可进行打单操作，更不可发货；
    
3.  若审核状态解除后，需要开发者明确提醒商家此订单已解除审核，请及时发货。
    

**后台：**

1.  需在数据库中新增该字段，未上线时可认为该字段为非必填字段，无该字段时则非审核订单，该字段值为1时才是审核订单。
    
2.  新增订单审核状态，开发者需于2019年11月30日前开发改造完成，避免平台将审核订单透出后开发者及商家无法识别。
