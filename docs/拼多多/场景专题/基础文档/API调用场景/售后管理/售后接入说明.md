# 售后接入说明

更新时间：2021-05-28 16:39:28

### 

业务简介

从买家发起售后申请开始，系统会为该订单生成一个售后单，根据此售后单可以持续跟踪售后详情。第三方开发者（ISV）和自研商家通过开放平台售后API获取到店铺的售后单数据，并把获取到的售后单信息在软件中展示给商家，供商家做售后处理。

### 

相关API介绍

-   [pdd.refund.list.increment.get 售后列表接口](https://open.pinduoduo.com/application/document/api?id=pdd.refund.list.increment.get)
    

根据售后单更新时间获取增量售后单，获取所有状态产生变动更新的售后单

单次调用入参更新时间区间30分钟，建议实时获取当前售后单时使用

-   [pdd.refund.information.get 售后单详情接口](https://open.pinduoduo.com/application/document/api?id=pdd.refund.information.get)
    

根据售后单号及订单号获取单笔售后单详情

已知订单号的情况下获取订单详情，可配合消息推送使用

-   [pdd.refund.agree同意退款](https://open.pinduoduo.com/application/document/api?id=pdd.refund.agree)
    

商家同意退款在以下售后状态可调用同意退款接口：【after\_sales\_status】

【2】买家申请退款，待商家处理

【3】退货退款， 待商家处理

【6】驳回退款，待用户处理

【7】已同意退货退款,待用户发货

【8】平台处理中

【31】已同意拒收退款，待用户拒收

-   [pdd.refund.returngoods.agree商家售后同意退货](https://open.pinduoduo.com/application/document/api?id=pdd.refund.returngoods.agree)
    

商家同意退货在以下售后状态可调用同意退货接口：【after\_sales\_status】

【2】买家申请退款，待商家处理

【6】驳回退款，待用户处理

【8】平台处理中

-   [pdd.refund.exchange.shipping商家换货发货](https://open.pinduoduo.com/application/document/api?id=pdd.refund.exchange.shipping)
    

商家同意换货发货时用此接口回传换货物流单号：【after\_sales\_status】

【3】退货退款，待商家处理

【7】已同意退货退款,待用户发货

【8】平台处理中

**已对接同意退款接口的开发者，以下退货入库与取消发货接口可不对接**

**（同意退款可真实触发退款，以下退货入库与取消发货接口只可在部分场景触发退款）**

-   [pdd.nextone.logistics.warehouse.update 退货入库](https://open.pinduoduo.com/application/document/api?id=pdd.nextone.logistics.warehouse.update)（可选）
    

用户申请退货退款场景，商家同意退货用户退回货品回传退货物流单号，商家收到退货确认入库后调用此接口告知平台退货入库成功。

配合在店铺后台配置售后小助手“退货成功自动退”模板，可触发同意退款

-   [pdd.rdc.pddgenius.sendgoods.cancel 取消发货](https://open.pinduoduo.com/application/document/api?id=pdd.rdc.pddgenius.sendgoods.cancel)（可选）
    

用户申请仅退款场景，商家未操作发货，确认货品拦截后可调用此接口告知平台取消发货成功。

配合在店铺后台配置售后小助手“取消发货自动退”模板，可触发同意退款

### 

售后单获取实施方案

#### 

方案一

使用[pdd.refund.list.increment.get](https://open.pinduoduo.com/application/document/api?id=pdd.refund.list.increment.get)售后列表接口实时获取店铺新增或发生更新的售后单，此接口不仅能够获取到新生成的售后单，还可以获取到订单信息有任何更新的售后单，此接口单次入参时间间隔较短为30分钟，在使用时需要定时轮询以获取最新的订单详情。

#### 

方案二

1.  通过消息推送服务客户端实时监听售后单变更消息
    
2.  再通过[pdd.refund.information.get](https://open.pinduoduo.com/application/document/api?id=pdd.refund.information.get)售后单详情接口获取售后单信息
    

消息推送对接请查看开放平台[《消息推送使用指南》](https://open.pinduoduo.com/application/document/browse?idStr=D3B6DBF3A5CB57E5)以及[售后消息](https://open.pinduoduo.com/application/document/message?secondCatId=4&id=9)。
