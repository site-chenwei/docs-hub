# 多多客使用指南

更新时间：2020-06-30 15:12:01

### 

**一、基础API** 

#### 

**1.1 接口简介**

基础API主要分为3块，分别为：商品类API、推广类API、订单查询类API

其中，

1）商品类API包括：

-   商品检索 
    
-   商品详情查询
    

2）推广类API包括：

-   生成推广位
    
-   生成推广链接
    

3）订单查询类API为：

-   增量查询订单接口。
    
-   查询已生成的推广位
    

#### 

**1.2 商品类API**

主要用于商品搜索以及查询商品具体详情信息。

**1.2.1 pdd.ddk.goods.search** **（多多进宝商品查询）**

本接口用于检索多多进宝商品。可根据筛选条件过滤出满足条件的商品，也可根据排序条件对检索出来的商品进行排序。

a、支持入参筛选条件有：

-   某一个标签id的商品、某一个分类id的商品。两者的区别在于：标签是一个商品有多个同级标签，例：某商品既属于一级标签服饰，也属于一级标签男装。 分类是一个商品仅有一个同级分类。获取标签id/分类id的方法，请查看以下接口: 商品API的pdd.goods.cats.get（商品标准类目接口）or pdd.goods.opt.get（查询商品标签列表）
    
-   标题包含某个关键词的商品
    
-   拼团价/券后价区间的商品
    
-   佣金比例/佣金金额区间的商品
    
-   优惠券金额区间的商品
    
-   某一时间段加入多多进宝的商品
    
-   销量区间的商品
    
-   是否只查看有优惠券商品
    
-   店铺dsr区间的商品
    
-   店铺类型（1-个人，2-企业，3-旗舰店，4-专卖店，5-专营店，6-普通店）
    

b、支持入参排序条件有：

-   佣金比例排序
    
-   拼团价/券后价排序
    
-   销量排序
    
-   优惠券排序
    
-   加入多多进宝时间排序
    
-   佣金金额排序
    
-   店铺dsr排序
    

注：渠道可以根据店铺数据（**店铺类型、店铺dsr**），也可以通过商品数据（ **商品销量、商品评价** ）去甄选优质的商品。

**1.2.2 pdd.ddk.goods.detail（多多进宝商品详情查询）**

本接口用于查询某个商品的详情信息：

-    商品标题
    
-    商品描述
    

商品分类 / 标签

最小拼团价、最小单买价

商品历史累计售卖件数

优惠券金额、优惠券数量、优惠券剩余数量、优惠券门槛、优惠券有效期

商品评价分、商品评价数量

佣金比例

商品轮播图

商品主图。该字段为长方形图片，用于首页

店铺dsr

店铺名称

**1.3 推广类API**

主要用于多多客为自己某个推广位生成某个商品的推广链接。

**1.3.1 pdd.ddk.goods.pid.generate（创建多多进宝推广位）**

本API用于多多客创建自己的推广位，推广位的用途：

-   与自己的用户体系进行关联，做代理模式。
    

举例：多多客为推手用户A生成一个pid =1\_1 , 为推手用户B生成pid=1\_2。若A是B的上级，当用户通过pid=1\_2的推广链接买了商品。由于用户和pid进行了映射关系，所以可以实现结算部分佣金给到A。

-   用于识别各投放资源位效果。
    

举例：将投放至群A的推广链接由pid=1\_1生成，投放至群B的推广链接由pid=1\_2生成。由于订单查询接口可以区分订单由某个pid推广产生，因此多多客可以统计查看群A和群B的推广效果数据。

**1.3.2 pdd.ddk.goods.promotion.url.generate（多多进宝单品推广链接生成）**

1）目前支持生成单人团商品推广链接和双人团推广链接。2者的区别是:

-   单人团是用户可以无需拼团，只接用拼团价购买商品
    
-   双人团是用户开团后分享给好友参团，好友参团后推手可获得双份佣金
    

2）推广链接类型有3种：普通链接、唤起微信app链接、唤起拼多多app链接。其中，

-   普通链接用于微信内环境使用
    
-   唤起微信app链接用于非微信内环境。用户在非微信环境点击该链接，会唤起微信客户端并访问相应H5推广页面。
    
-   唤起拼多多app链接用于非微信内环境。用户在非微信环境点击该链接，若用户装有拼多多App，会唤起拼多多App并访问相应H5推广页面；若用户没有装App，会在当前App打开拼多多H5推广链接。

**1.4 查询订单类API**

主要用于追踪多多客查询推广位带来的订单以及订单状态更新

**1.4.1 pdd.ddk.order.list.increment.get（最后更新时间段增量同步推广订单信息）**

通过本接口，多多客可以定时同步有状态更新（包括创建订单）的订单到本地库。

1）订单信息主要包括：

-   订单状态以及各状态变更时间
    
-   订单支付金额、佣金比例以及金额
    
-   订单所属商品编号、标题、缩略图
    

2）订单状态包括：

-   未支付：订单创建还未支付
    
-   已支付
    
-   已成团：单人团支付成功后会状态会马上置为成团；双人团支付后需有其他人参团状态才会置为成团
    
-   确认收货
    
-   审核成功：确认收货15天后若未发生售后，订单状态会置为审核成功
    
-   审核失败（不可提现）：若订单发生售后，订单状态会置为审核失败
    
-   已经结算：每月20号会结算当月15号及以前审核通过的订单，状态并置为已结算
    
-   非多多进宝商品（无佣金订单）：用户访问推广链接时，该商品不在多多进宝推广计划中，因此购买商品产生的订单为非多多进宝订单。
    
-   已处罚：当判定某个pid在拼多多主站站内导流，该pid所有的订单均不会结算佣金，并会置为已处罚状态。
    

**1.4.2 pdd.ddk.goods.pid.query（查询已经生成的推广位信息）**

该接口用于查看多多客所有已生成的pid（推广位编号）以及推广位名称

**二、高级API**

#### 

**2.1 高级单品推广API**

高级单品推广是商家为特定多多客设置的推广计划，可设置特定的佣金比例和优惠券，高级单品推广分为专属推广以及招商推广。

**2.1.1 专属推广**

1）专属推广计划介绍

-   商家可在商家后台为多多客设置专属佣金比例和优惠券金额。
    
-   用户访问设有定向计划的商品推广链接l 若链接所属多多客在定向计划内，则用户可领取计划中的优惠券，用户购买此商品，将以计划中的佣金比例给推手结算。
    
-   若链接所属多多客没在定向计划中，则用户仅可领取通用优惠券，且用户购买后，以通用佣金比例给推手结算。
    
-   工具商可将定向推广计划赋能给授权者。即工具商的多多客id加入定向推广计划后，那么授权他的多多客也可享受该计划权益（佣金和优惠券）
    
-   当定向推广计划优惠券领完，用户可领取通用券。
    

2）生成专属推广链接方法

-   通过 **pdd.ddk.direct.goods.query（定向推广商品查询接口）**获取自己的专属推广商品信息。
    
-   通过 **pdd.ddk.goods.promotion.url.generate（多多进宝推广链接生成）**即可生成自己的专属推广链接。
    

**2.1.2 招商推广**

1）招商推广计划介绍

-   商家为多多客创建招商推广计划后，其他人通过转链API或生成推广链接API，即可享受该招商推广计划的佣金比例和优惠券金额。
    
-   当招商推广计划中优惠券领完，用户可领取通用券。
    

2）招商推广计划获得方法

方法一：

-   获取招商多多客ID后，可以通过 **pdd.ddk.zs.unit.goods.query（查询招商推广计划商品）** 获取该招商多多客所有的招商推广计划
    
-   获得招商多多客ID后，通过**pdd.ddk.goods.promotion.url.generate（多多进宝推广链接生成）**入参增加 zs\_duo\_id（**招商多多客ID**），即可生成招商推广计划。
    

方法二：

-   获得招商多多客分享出来的招商推广链接
    
-   通过 **pdd.ddk.goods.zs.unit.url.gen（多多进宝转链接口）**，可以将其他人的招商推广链接转换为自己的。

**2.2 营销工具推广类API**

营销工具推广类API帮助渠道在不同场景下进行推广，节省渠道选品工作，提升流量转化率。支持生成唤起微信App推广链接、唤起拼多多App推广链接。

**2.2.1 分享红包赚佣金**

1）活动介绍

活动利益点

-   推手无需批量选品，一键即可生成红包链接，轻松分享，活动页面内所有单品产生的有效订单均可结算佣金！
    
-   页面单品推荐千人千面，基于用户行为智能推荐与买家兴趣点匹配的单品，有效提升订单转化率，实现精准营销。
    
-   红包自带互动性和趣味性，能有效刺激用户点击与分享，提升用户活跃度，快速形成裂变。
    

**活动规则**

-   每个用户拆开红包后，使用有效期为2小时。
    
-   单个红包链接领取人数不限，但每个用户同一个链接一天仅可领一次。
    
-   同一用户可领取不同红包链接中的红包。
    
-   红包推广和单品推广的结算逻辑：结算给用户购买商品前最后点击的领券页对应的推手pid。
    

2）使用方法

-   通过 **pdd.ddk.rp.prom.url.generate（生成红包推广链接）**即可生成红包推广链接
    
-   本API为特权接口，日均1000单渠道可联系工作人员申请此权限
    

**2.2.2 cms商城推广**

**1）活动介绍**

-   仅需将商城推广链接分享给用户，用户购买商城任一商品，均会结算相应佣金给你。
    
-   拼多多运营人员选品，免去选品烦恼
    
-   每日拼多多运营人员精选当天高佣优质商品，置顶推荐在商城首页，大大节省你选品的工作。
    
-   商品丰富，用户购买转化率高
    
-   商城推广页汇聚百万级多多进宝商品，用户容易找到心仪的商品
    

 **2）使用方法**

通过 **pdd.ddk.cms.prom.url.generate（生成商城推广链接）**即可生成cms推广链接

2.2.3 主题推广

**1）活动介绍**

渠道可以获取拼多多官方运营或其他推广者创建的主题活动，通过API生成主题活动推广链接进行推广。无需选品，只要将活动推广链接放置在你的某个流量位即可。

备注：主题推广API也支持你获取主题商品后选品再编辑哦~

**2）使用方法**

方法一：直接分发推广链接

-   通过 **pdd.ddk.theme.list.get（多多进宝主题列表查询）**获取想要推广的主题id
    
-   **pdd.ddk.theme.prom.url.generate（多多进宝主题推广链接生成）** 入参传入主题id以及pid，即可生成相应主题的推广链接。
    
-   目前主题推广链接暂不支持小程序格式，小程序开发者可以自己获取主题商品后自行开发小程序主题推广页。
    

方法二：获取主题商品后选品再编辑

-   通过 **pdd.ddk.theme.goods.search（多多进宝主题商品查询）**入参传入主题id，获取某个主题下的商品。可以根据你的具体需求筛选想要的主题商品开发自己的主题页。

2.3 多多客工具API

**2.3.1 功能简介**

-   多多客工具API是针对工具商特别定制，工具商获取多多客授权后，即可帮多多客完成创建推广位，查询推广位产生的订单等操作。 
    
-   用户可自行前往多多进宝网站完成提现，第三方无需开发结算系统。
    

**2.3.2 授权流程**

-   请在【拼多多开放平台—开放管理—应用管理】，添加你的回调地址。开放网址：[https://open.pinduoduo.com/application/app/list](https://open.pinduoduo.com/application/app/list)
    
-   授权方法详见拼多多开放平台文档，文档网址：[https://open.pinduoduo.com/application/document/browse?idStr=BD3A776A4D41D5F5](https://open.pinduoduo.com/application/document/browse?idStr=BD3A776A4D41D5F5)
    

**2.3.3 工具商独特玩法**

**1）赋能授权多多客**

工具商可以将自己的能力赋能给授权者。即，工具商具备特权接口调用能力以及有专属推广计划，授权多多客也能具备同样特权。特权包括但不仅限于如下：

-   高佣权限
    
-   红包生成API
    
-   订单拉新API
    

**2）工具商跑单平台补贴**

平台每月都有订单补贴活动，当工具商下面的授权多多客总单量达到要求，我们会将补贴奖励结算给工具商。

**三、 FAQ**

Q1：“推客不存在”或“50001未知错误”是什么原因？

A：这种情况大多是是没有绑定多多进宝api，请进入多多进宝官网进行client\_id绑定。

Q2： 如何申请直接跳转到拼多多小程序？

A ：申请方式请看：[https://jinbao.pinduoduo.com/message/detail?crumbs=broadcast&id=200](https://jinbao.pinduoduo.com/message/detail?crumbs=broadcast&id=200)

Q3：多多进宝绑定开发平台client\_id后，可以解绑么？

A：暂不支持解绑，所以绑定时请谨慎操作。
