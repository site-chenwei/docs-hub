# 实物卡券对接指南

更新时间：2021-06-28 15:37:32

一、适用范围

实物卡券（goods\_type=1）适用于同时支持到店自提和物流发货的实物商品。该卡密将在用户下单成交后由平台制码发给用户。该券码作为凭证，线下实际使用时，由用户把平台生成的券码提供给商家，商家完成服务后将券码核销，交易完成。若券码未核销，则券码过期后会自动给用户退款。

商家自研接入开平请申请“电子凭证商家后台系统”这个应用类型。

注：必须有店铺招商对接

二、操作指南

1、申请权限

如需开通实物卡券售卖，请先联系拼多多侧对接的招商同事进行确认，申请开通相关白名单。

2、商品发布

关键参数：

1） 选择商品类型：发布商品时，商品类型需选择“普通商品”（goods\_type=1）。（注：需要先确定发布的类目，仅部分类目支持实物卡券售卖，如类目不满足要求，可联系招商同事沟通）

2） 选择发货方式：发货方式选择“电子交易凭证”，如该商品需同时支持物流发货，可同时选择“物流配送”

![](https://t16img.yangkeduo.com/perse-open-api/2021-03-29/c673f97f-ee05-4218-9abd-de01932f75a1.png)

3） 设置是否支持指定门店：如需支持用户下单时指定门店提货，需要在“本地服务”中勾选“支持指定门店自提”，并选择相应的门店组

4）设置电子凭证信息：需设置电子凭证有效期（支持固定起止时间、发货后至固定时间、发货后N天 三种模式），请设置合理的有效期范围，过期未使用的卡券平台将自动给用户退款

2、订单创建

消费者拼单成功后，拼多多平台将进行制码并给用户发放。券码发放后，消费者可在订单列表或订单详情页，点击“查看卡券”，可以查看卡券及其相关信息，包括卡券状态、有效期及使用说明（使用说明需要联系招商单独配置）。

3、获取订单

商家可以通过订单增量接口（pdd.order.number.list.increment.get，非必须），实时获取拼多多侧的订单状态、售后状态等信息。

4、核销券码

在完成履约后，可通过商家后台（BAPP或MMS）对券码进行核销，或者通过接口回调核销（需要找招商配合申请接口白名单权限）。

1） 商家后台核销：通过商家后台核销工具模块，扫描或输入券码 进行核销。

2） 接口核销：

1、通过券码查询接口（pdd.voucher.ota.card.prepare.verification）校验券码对应商品及券码是否可用

2、通过券码核销接口（pdd.voucher.ota.card.verification）将券码进行核销

（注：售后中订单无法核销，需要用户取消售后或商家驳回售后至售后单关闭后才能核销成功，务必确保券码及时成功核销）

5、售后处理

可通过商家后台人工处理售后或者通过开平接口处理售后。

1）售后列表接口（pdd.refund.list.increment.get，非必须）

使用场景：拉取用户的售后申请列表信息，以便能自动处理可以同意售后的订单。

请求参数说明：

实物卡券与普通实物商品无差别

返回参数说明：

主要关注以下参数：

order\_sn、id（售后id）、after\_sale\_reason、refund\_amount、order\_amount、total\_count（判断有多少分页）

2）同意售后接口（pdd.refund.agree，非必须）

使用场景：在拉取售后列表后，商家内部如判断该订单能同意退款，可通过该接口回调拼多多侧同意退款。对于需要驳回的退款申请，还是需要在商家后台人工处理。
