# 商家卡券对接指南

更新时间：2021-06-28 15:37:32

### 

一、适用范围

商家卡券（goods\_type=15）功能可以帮助商家实现系统自动下发卡密，卡密信息由商家提供上传到拼多多店铺后台，用户使用后需要商家及时同步回传卡密状态。

商家卡券类商品，拼团成功后系统将自动下发商家预存的卡密，每笔订单只支持下发一个卡密 ，用户核销后默认T+7结款，未核销的卡密过期后将给用户自动退款。

商家自研接入开平请申请“电子凭证商家后台系统”应用类型。

注：必须有店铺招商对接

### 

二、操作指南

1、申请权限

如需开通商家卡券售卖，请先联系拼多多侧对接的店铺招商进行确认，申请开通相关白名单。

2、商品发布

关键参数：

1）选择商品类型：发布商品时，商品类型需选择“卡券（商家定义）”（goods\_type=15）。（注：需要先确定发布的类目，并申请开通白名单后才能看到该商品类型）

![](https://t16img.yangkeduo.com/perse-open-api/2021-03-29/3a9bea7f-adfa-4975-aaf7-76e59d4c31e4.png)

2）添加卡密库存：

a、通过商家后台添加，编辑价格和库存时，点击“添加卡密”，上传卡密（卡号+密码格式，不要重复），注意要如实填写库存信息，确保用户下单后，商品可以正常发货，避免可能的物流违规。

b、也可通过开平批量添加卡券接口（pdd.voucher.virtual.card.batch.add）批量添加卡券。

使用该接口添加卡券前，需要先发布商品，记录下拼多多侧生成的商品ID（goods\_id）和SKU ID（sku\_id）。可在商品管理-商品列表中查看商品ID，点击“修改价格”可查看SKU ID。接口批量添加完卡密后，需要去商家后台修改商品库存。

3）设置电子凭证信息：需设置电子凭证有效期（支持固定起止时间、发货后至固定时间、发货后N天 三种模式），请设置合理的有效期范围，过期未使用的卡券平台将自动给用户退款

3、订单创建

消费者拼单成功后，拼多多平台将把商家预存在平台的卡密自动下发给用户。券码发放后，消费者可在订单列表或订单详情页，点击“查看卡券”，可以查看卡券及其相关信息，包括卡券状态、有效期及使用说明（使用说明需要联系招商单独配置）。

4、获取订单

商家可以通过订单增量接口（pdd.order.number.list.increment.get，必须），实时获取拼多多侧的订单状态、售后状态、订单与券码的绑定关系等信息（注：只有已发货的订单才能拉取到券码信息）。

5、核销回调

在用户使用卡券时，商家需要通过卡券核销回调接口（pdd.voucher.virtual.card.verification，必须）及时回传核销信息，告知拼多多卡券已核销。（注：售后中订单无法核销，需要用户取消售后或商家驳回售后至售后单关闭后才能核销成功，商家需要重试直至成功）

6、售后处理

可通过商家后台人工处理售后或者通过开平接口处理售后。

1）售后列表接口（pdd.refund.list.increment.get，非必须）

使用场景：拉取用户的售后申请列表信息，以便能自动处理可以同意售后的订单。

请求参数说明：

after\_sales\_status只需要关注“2：买家申请退款，待商家处理”这一场景

after\_sales\_type只需关注“2：仅退款”这一场景

卡券类商品无实物（实物卡券除外，与普通商品无差别），退款时不涉及退货。卡券已核销视为订单已完结，需要联系商家或者平台客服处理。

返回参数说明：

主要关注以下参数：

order\_sn、id（售后id）、after\_sale\_reason、refund\_amount、order\_amount、total\_count（判断有多少分页）

2）同意售后接口（pdd.refund.agree，非必须）

使用场景：在拉取售后列表后，商家内部如判断该订单能同意退款，可通过该接口回调拼多多侧同意退款（注意：同意退款前务必先作废券码，以免产生资损）。对于需要驳回的退款申请，还是需要在商家后台人工处理。
