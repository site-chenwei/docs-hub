# 实时卡券对接指南

更新时间：2021-06-28 15:37:32

### 

一、适用范围

实时卡券（goods\_type=24）适用于商家自有卡密进行发货的场景。该卡密将在用户下单时平台实时请求商家获取卡密，商家需要在消费者使用卡密时实时回传核销结果。

商家自研接入开平请申请“电子凭证商家后台系统”这个应用类型。

注：必须有店铺招商对接

### 

二、操作指南

1、申请权限

如需开通实时卡券售卖，请先联系拼多多侧对接的招商同事进行确认，申请开通相关白名单。

2、商品发布

关键参数：

1）选择商品类型：发布商品时，商品类型需选择“卡券（商家定义 & 实时）”（goods\_type=24）。（注：需要先确定发布的类目，仅部分类目支持实时卡券售卖，并申请开通白名单后才能看到该商品类型）

![](https://t16img.yangkeduo.com/perse-open-api/2021-03-29/819eca5a-ab3c-47cf-a869-170450e3ab00.png)

2）配置外部sku编码：该编码会在制码时传给商家，商家依据该编码识别需制码的商品种类。

3）设置电子凭证信息：

a、需选择电子凭证类型（仅可兑换一次或可多次兑换）

b、需设置电子凭证有效期（支持固定起止时间、发货后至固定时间、发货后N天 三种模式），务必保证卡券仅在配置的有效期内可使用，过期未使用的卡券平台将自动给用户退款

3、订单创建

消费者拼单成功后，拼多多平台将通过商家配置的方舟制码接口（pdd.voucher.realtime.apply.make.code，必须）实时请求商家制码。商家制码成功后，消费者可在订单列表或订单详情页，点击“查看卡券”，可以查看卡券及其相关信息，包括卡券状态、有效期及使用说明（使用说明需要单独配置）。

4、获取订单

商家可以通过订单增量接口（pdd.order.number.list.increment.get，非必须），实时获取拼多多侧的订单状态、售后状态等信息。

5、核销回调

在用户使用卡券时，商家需要通过卡券核销回调接口（pdd.voucher.realtime.verify.sync，必须）实时回传核销结果，告知平台卡券已核销。（注：售后中订单无法核销，需要用户取消售后或商家驳回售后至售后单关闭后才能核销成功，商家需要重试直至成功）

6、售后处理

可通过商家后台人工处理售后或者通过开平接口处理售后。

1）售后列表接口（pdd.refund.list.increment.get，非必须）

使用场景：拉取用户的售后申请列表信息，以便能自动处理可以同意售后的订单。

请求参数说明：

after\_sales\_status只需要关注“2：买家申请退款，待商家处理”这一场景

after\_sales\_type 只需关注“2：仅退款”这一场景

卡券类商品无实物（实物卡券除外，与普通商品无差别），退款时不涉及退货。卡券已核销视为订单已完结，需要联系商家或者平台客服处理。

返回参数说明：

主要关注以下参数：

order\_sn、id（售后id）、after\_sale\_reason、refund\_amount、order\_amount、total\_count（判断有多少分页）

2）同意售后接口（pdd.refund.agree，非必须）

使用场景：在拉取售后列表后，商家内部如判断该订单能同意款，可通过该接口回调拼多多侧同意退款（注意：同意退款前务必先作废券码，以免产生资损）。对于需要驳回的退款申请，还是需要在商家后台人工处理。
