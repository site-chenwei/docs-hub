对接主要分为三大部分：准备阶段、生产测试、线上投产；针对每部分具体说明如下：

2.2 对接流程介绍  

#### 

• 拼多多推送虚拟商品订单（收单）：  

当用户创建虚拟商品新订单后，  拼多多调用商家提供的充值接口，商家同步返回是否接到推送；  

商家自己异步的去进行充值；商家提供的充值接口格式具体详见文档中订购流量、订购话费、订购视频和会员；

备注：该流程商家只要成功接到订单信息并通过基础校验，则返回成功，请不要添加额度等限制拦截。  

店铺异步充值过程中出现问题，再回调拼多多平台充值失败；  

#### 

• 商家回调拼多多的发货接口（上报充值结果）：  

当商家对订单充值操作完成后，不管成功还是失败，都要将充值结果组装成请求，通过开发平台调用拼多多的发货接口，完成回调；最后我们根据回调结果，进一步推进订单状态变化  

如果充值成功，我们则完成订单的发货和确认收货。如果充值失败，我们则发起退款申请；  

• 商家开放平台：  

商家接入开放平台的具体信息，详见开放平台官网：https://open.pinduoduo.com  

#### 

• 详细流程说明：  

 接口超时说明：  

1） 拼多多调用商家发货接口（参考上图 HTTP 请求 1）:   

超时时间：5000ms  

备注：如请求超时或其他原因等导致的请求失败，拼多多端会定时发起重试请求，直到正确接收到商家端的返回结果为止，此处商家端服务接口需做好幂等控制  

2） 商家异步回调发货结果接口（经过开放平台，参考上图 HTTP 请求 2）：超时时间：3000ms  

备注：如请求超时或其他原因等导致的请求失败，则商家端需发起重试请求，否则订单可能无法正常收发货

3 拼多多调用商家接口说明  

## 

3.1 同步接口及协议  

接口统一采用 GET 请求接收订单，返回数据格式为 Json ；目前仅支持 HTTPS  注意：数据字符编码均采用 UTF-8 编码

3.2 TOKEN 方式

### 

3.2.1 加密  

加密参数  

拼多多给代理商家生成一个特定的密钥 token，代理商家自行保存好，不需要参与传送  

##### 

加密签名生成规则如下：  

第一步：将所有 URL 中的参数值的参数按照参数名 ASCII 码从小到大排序 （ 字 典 序 ） ， 使 用 URL 键 值 对 的 格 式 （ 即

para1=val1&para2=val2&para3=val3…）拼接成字符串 tempStr；参数规则：  

1. 参数名 ASCII 码从小到大排序（字典序）  

2. sign 不参与签名  

3. HTTP 头部参数名不参与签名  

4. 签名参数名区分大小写  

第二步：在 tempStr 最后拼接上 key 和 signType 得到待签名字符串 signStr，并对 signStr 进行 MD5 运算，得到 sign 值 signValue  

          例如：  

推送请求参数(Request Param)：  

signType=md5&amount=1&mctNo=100&mobile=1391111111&notify

Url=https://gw-api.pinduoduo.com/api/router&outOrderNo=3012-1622547876323&parPrice=100  

按照 ASCII 码从小到大排序后(orgAscIIStr)：  

amount=1&mctNo=100&mobile=1391111111&notifyUrl=https://gw-api.pinduoduo.com/api/router&outOrderNo=3012-

1622547876323&parPrice=100&signType=md5  

假设商家 签名秘钥 token：123456789   

签名原串字符如下(orgAscIIStr&key=123456789)：  

amount=1&mctNo=100&mobile=1391111111&notifyUrl=https://gw -api.pinduoduo.com/api/router&outOrderNo=30121622547876323&parPrice=100&signType= md5&key=123456789

签名 Sign（对上步生成的签名原串字符做 MD5 签名）:  

Md5(amount=1&mctNo=100&mobile=1391111111&notifyUrl=https: 

//gw-api.pinduoduo.com/api/router&outOrderNo=3012- 

1622547876323&parPrice=100&signType=md5&key=123456789)  上述 MD5 值为： f3ecf6e518656d8574c8b50957f054e5  

将 sign=signValue 附加在调用接口的参数中

import java.security.KeyFactory; import java.security.interfaces.RSAPublicKey; import java.security.spec.X509EncodedKeySpec;

import javax.crypto.Cipher;

import org.apache.commons.codec.binary.Base64;

public class RSAUtil {     private static final String ALGORITHM = "RSA";

    private static final String CIPHER\_ALGORITHM = "RSA/ECB/PKCS1Padding"; 

    private static final Base64 DECODER = new Base64();

    /\*\* 

\* 公钥解密

\* @param data

\* @param sourceKey

\* @return 

\* @throws Exception

     \*/

    public static String decryptByPublicKey(String data, String sourceKey) throws Exception {

        X509EncodedKeySpec x509KeySpec = new X509EncodedKeySpec(DECODER.decode(sourceKey));         java.security.Security.addProvider(new org.bouncycastle.jce.provider.BouncyCastleProvider());

        RSAPublicKey publicKey = (RSAPublicKey)

KeyFactory.getInstance(ALGORITHM).generatePublic(x509KeySpec);

        return decryptByPublicKey(data, publicKey);

    }

    public static String decryptByPublicKey(String data, RSAPublicKey publicKey) throws Exception {

        Cipher cipher = Cipher.getInstance(CIPHER\_ALGORITHM);         cipher.init(Cipher.DECRYPT\_MODE, publicKey);

        byte\[\] b1 = DECODER.decode(data);

        /\*\* 执行解密操作 \*/

        byte\[\] b = cipher.doFinal(b1);

        return new String(b); 

    }

}
