# 电子面单接入指南

更新时间：2021-09-19 15:47:24

### 

一、前言

随着电子商务平台和物流服务信息化飞速发展，面单号(或称之为运单号)成为物流服务商串联快递单、订单、商家、商品等各种信息的枢纽。相比之下，传统纸质面单价格高、信息录入效率低、信息安全隐患等方面的劣势已愈发凸显，电子面单作为多种信息的载体则逐渐成为行业主流。

2. 对接流程

2.2 电子面单相关业务流程

1) 同步商家开通状态和账户余额情况：使用[pdd.waybill.search](https://open.pinduoduo.com/application/document/api?id=pdd.waybill.search)获取商家开通电子面单情况， 可以引导商家进入[https://mms.pinduoduo.com/logistics/open](https://mms.pinduoduo.com/logistics/open)进行申请，对应快递公司网点在接收到申请后将会联系商家进行电子面单账号开通，随后根据线下约定对账户进行充值。

账户字段中，quantity标识了面单余额，cancel\_quantity标识取消的电子面单数量，即主动取消的电子面单数量，字段allocated\_quantity为已经取号的面单数量，recycled\_quantity为已经回收的面单数量。

2) 电子面单取号：取号前做如下前置判断：1）商家所选快递公司下，对应账户余额充足，直营类无需判断，在waybill.get 接口入参时，

标识扣款账户的 user\_id（即店铺 mall\_id 同授权接口的 owner\_id，pdd 面单允许 A

店铺使用 B 店铺账号进行打单）2）取号时发货地址设置为账户网点对应发货地址，注意，使用 search 接口进行获取。

取号时，注意以下逻辑：

-   单个面单号获取，订单 list 中最多支持200 个订单关联，其中 200 个订单不能重复；
    
-   批量取号场景下，最多不超过 10 个面单取号，如果其中一个面单取号失败，直接返回报错，不减扣余额；
    
-   已经成功取号的订单号不能重复出现在多个面单取号中；
    
-   对于已经取号后的订单重新取号，会返回已经取号的原面单号；
    
-   trade\_order\_info\_dtos.order\_channels\_type
    
    传值枚举如下PDD（拼多多），TB（淘宝），TM（天猫），JD（京东），ALBB（阿里巴巴），YZ（有赞），WD（微店），MGJ（蘑菇街），YJ（云集），BB（贝贝网），ZZ（转转），KS（快手小店），DD（当当网），XMYP（小米有品），SK（寺库），JM（聚美优品），MY（蜜芽），XHS（小红书），MT（萌推），WPH（唯品会），PP（拍拍），EBAY（ebay），AMAZON（亚马逊），SN（苏宁），GM（国美），YHD（1号店），VANCL（凡客），YL（邮乐），YG（优购），LF（乐蜂），JS（聚尚），PX（拍鞋），YT（银泰），OTHERS("其他")。
    

3) 电子面单号取消：已经取号的电子面单支持进行主动取消回收，主动取消的面单将会在 15 天内无轨迹进行回收，如果面单没有取消且30天无轨迹，那么面单也将会被回收。取消单号存在取消限制，对已经取号的面单，如果取消单量超过已取号数量的 20%，将无法继续取消。

4) 电子面单查询：支持电子面单查询，返回电子面单打印信息字段。

5) 电子面单更新：已取号面单更新，发货人仅支持姓名和联系方式修改，无法修改地址；订单部分字段无法进行修改。

三、常见问题

1. 电子面单取号 get 接口，请求报错：“同一个交易订单不能申请多个网点的面单”

【问题原因】：

原因是订单号已经取过面单号，且再次取号时发货地址发生变化，匹配到其他网点。

【排查方式】：

请检查该订单是否已经取过号了，且再次取号时，发货地址或者物流公司编码（wp\_code）与第一次取号时不同；

【解决方法】：

1) 如果要更改发货地址或者快递公司，请调用 cancel 取消接口后再重新取号；

2) 若没找到取号记录，请修改包裹 id（package\_info.id），再取号；

2. 调用电子面单取号接口，报错：“面单订购服务不存在”。

【问题原因】：

说明该用户未订购取号时所使用的物流商。

【排查方式】：

1) ISV请通过订购关系查询接口查询商家账号在所用 WP 下所有的订购关系，查看返回报文中是否存在所使用 WP 对应的订购关系（若不存在订购关系，返回会为空）；

2) 商家请使用取号账号登录拼多多商家后台（https://mms.pinduoduo.com/logistics/open），检查取号时使用物流商是否存在订购关系，订购关系是否已被审核通过；

3. 调用电子面单取号接口，报错：“发货地址没有匹配的电子面单服务”。

【问题原因】：

说明发货地址填写错误，用户在该地址下无订购关系，需要使用申请面单服务时候注册的发货地址。

【排查方式】：

1) ISV请通过订购关系查询接口，查询商家账号在该物流商下所有的订购关系，检查取号时所用的地址是否在订购地址中。注意：是否包含某些特殊字符或者不可见字符、多空格少空格情况（地址必须完全一致），取号发货地址中 city 、detail、district、province、town 必须和订购关系查询接口返回的对应字段完全一致，没有就置空或不传 key 值；

2) 商家使用取号账号登录平台，检查取号时使用物流商、发货地址下，是否与订购关系列表中对应 WP 下订购所使用的地址一致。注意：是否包含某些特殊字符或者不可见字符、多空格少空格情况（地址必须完全一致）；

4. 调用电子面单取消接口，报错：“取消次数超过最大限制”。

【问题原因】：

商家对于没有使用的面单号，可以进行取消，但是取消有次数限制（取消总数不能超过当前网点取号总数的20%，取号总数不超过500单的不做此限制），建议不要随意取消；没有使用的面单号可以放任不管，快递公司会根据电子面单回收规则自动回收，商家也可自行联系网点回收。

5. 三段码为空。

【解决方法】：

收件人地址取拼多多订单接口中的地址不要做映射；详细地址取订单接口中 receiver\_address 字段；调用取号接口时单例调用以保持接口稳定。

6. 对接快递公司增值服务。

拼多多电子面单支持快递公司的增值服务，通过pdd.waybill.get接口中的logistics\_services字段传入。顺丰快递和京东配送按照以下方式传入

1) 顺丰快递

顺丰支持【时效产品】和【保价】两种增值服务，其中【时效产品】为必填项，【保价】为选填项。

【传值样例】

Plain Text收起

{

    "logistics\_services":{

    "TIMED-DELIVERY": {

    "value": "2"},//时效产品

    ”INSURE": {

    "value": "510.1"}//保价

    }

}

注意事项：

-   时效产品不传值则默认为“顺丰特快”；
    
-   【保价】服务需要传具体的报价金额，单位为“元”，精确到小数点后1位；
    
-   支持的时效产品可在pdd.waybill.search接口获取到；
    

2) 京东配送

京东配送支持【产品类型】一种增值服务，且是必填项。

【传值样例】

Plain Text收起

{

    "logistics\_services":{

    "PRODUCT\_TYPE": {

    "value": "2"}//时效产品

    }

}

7\. 对接快递公司增值服务。

7.1.接口变更

1) 快运电子面单与的快递电子面单云打印接口为同一接口，

所以接口的行为保持一致。每次传入一个包裹，返回一个子单号和对应的母单号。

2) 子母件模式不支持批量取号。如果需要取得连续递增的子单号，每次需要在同一个user\_id下传入相同的订单id列表，不同的包裹id， 如果订单列表相同，包裹id也相同，会触发幂等。

3) 子件面单除子单号不同，其他信息完全相同。

4) 入参新增包裹总件数字段trade\_order\_info\_dtos. package\_info. total\_packages\_count，快运公司需要在面单上体现子母件总包裹数，必填

5) 入参新增包装类型字段 trade\_order\_info\_dtos. package\_info. packaging\_description，快运包装方式描述，选填

6) 入参新增货品类型字段 trade\_order\_info\_dtos. package\_info. goods\_description，快运货品描述，选填

7) 接口返回数据中包含子母件的子单号waybill\_code和母单号parent\_waybill\_code

8) ISV需要注意，在快递模式下，返回的单号只有waybill\_code，waybill\_code是挂载物流详情和用来发货的单号，而在子母件中，这个对应的是parent\_waybill\_code，

而waybill\_code是子单号，不能用来发货。

9) 快运场景部分快递公司有必选的增值服务，若取号的时候不传则会取号失败；

7.2. 子母件取消接口行为

1) 不能单独取消子件

2) 取消母件则取消所有子件

7.3. 子母件更新接口行为

1) 子母件不允许更新
