# 订单合并发货场景说明

更新时间：2026-03-26 19:45:49

合单ID（open\_address\_id）是在收件人信息加密的背景下帮助商家/ISV进行合单判断的字段，若订单中open\_address\_id一致，可直接进行合单。合单ID可通过订单相关接口：pdd.order.information.get、pdd.order.list.get、pdd.order.number.list.increment.get获取。

合单ID变更说明：

open\_address\_id原窗口期为1天（即同收件人信息在同店铺的跨日订单open\_address\_id不一致）变更为7天，变更时间为每周日0点（即每周日到次周六，同收件人信息在同店铺的订单，open\_address\_id一致）。

open\_address\_id\_2：订单接口中新增 open\_address\_id\_2， open\_address\_id\_2生成方式逻辑与open\_address\_id一致，有效窗口期为7天，且变更时间错开，变更时间为每周四0点。

以上变更将于2025年3月3日上线。

举例：当同消费者在同店用同收件人信息分别于3月9日、3月13日、3月16日下单订单a、b、c，那么订单a、b的open\_address\_id一致，订单b、c的open\_address\_id\_2一致。

![](https://t16img.yangkeduo.com/perse-open-api/2025-02-26/51ae63d9-77e8-4777-89fd-6590839cb73b.jpeg)

合单方式说明：

1、有收件人信息无oaid订单：部分订单无open\_address\_id、open\_address\_id\_2，直接调用合单接口（pdd.order.merge.ship.order.group）判断。

2、有oaid订单：通过open\_address\_id、open\_address\_id\_2判断，当订单的任一open\_address\_id或 open\_address\_id\_2一致，则可直接合单。若3天以内\*订单open\_address\_id和open\_address\_id\_2均不相同，则不可合单，无需调用合单接口。

\*说明：如3月5日和3月9日的open\_address\_id和open\_address\_id\_2均不同，但有可能合单。

特别注意：请开发者合理调用，勿将所有订单轮询调用合并发货接口查询是否支持合单，请按照场景处理进行判断。

FAQ

1、同一笔订单的open\_address\_id或open\_address\_id\_2在不同时间通过订单接口获取，会不一样吗？

只有订单修改了收件人信息会不一样，否则同一笔订单的open\_address\_id或open\_address\_id\_2不会变更。

2、为什么同一收件人信息的两笔订单open\_address\_id和open\_address\_id\_2会存在都不相同的情况？

open\_address\_id和open\_address\_id\_2的变更周期为7天，若两笔订单的日期超过此间隔，则open\_address\_id和open\_address\_id\_2都不一致。
