# 提交应用上线审核前须知

提交应用上线审核前请仔细阅读文档且必须满足如下要求，否则不予审核通过。提交上线审核会在T+5个工作日内按序审核处理，审核通过或者驳回会有相应短信通知，或可在控制台-应用详情页面查看审核进度。

### **商家授权后跳转页面要求**

#### **【搬家上货、商品优化分析、打单、订单处理、进销存】授权场景要求**

| **授权场景** | **B/S架构** | **C/S架构** |
| --- | --- | --- |
| 服务市场进入 | 【必须】 授权登录跳转至软件内，根据接口获取的店铺ID和名称在自有账号体系中直接为店铺生成账号，达到登录状态，可直接使用功能 | 【必须】 跳转至回调页给出有效下载链接、加密处理的授权码（并提示用户勿泄露）、联系方式、必要的使用说明书（文档、图片、视频） |
| 软件内店铺绑定 | 【可选】 提供店铺授权入口，点击授权弹出授权页面，授权成功后显示店铺信息及授权状态等 | 【必须】 店铺授权绑定必须通过唤醒本地浏览器拼多多授权页面，用户手动配置完成绑定并显示店铺信息及授权状态等（不允许出现内嵌浏览器或者输入账密形式完成授权和登录店铺后台） |

#### **【ERP/WMS】授权场景要求**

| **授权场景** | **B/S架构** | **C/S架构** |
| --- | --- | --- |
| 服务市场进入 | 【必须】 授权登录跳转至软件登录页 可以不提供注册，线下联系分配账密 | 【必须】 跳转至回调页给出有效下载链接、加密处理的授权码（并提示用户勿泄露）、联系方式、必要的使用说明书（文档、图片、视频） |
| 软件内店铺绑定 | 【必须】 需提供店铺授权入口，点击授权弹出授权页面，授权成功后显示店铺信息及授权状态等 | 【必须】 店铺授权绑定必须通过唤醒本地浏览器拼多多授权页面，用户手动配置完成绑定并显示店铺信息及授权状态等（不允许出现内嵌浏览器或者输入账密形式完成授权和登录店铺后台） |

#### **两个授权场景说明及如何完成自测**

场景一：服务市场进入

用户服务市场订购，店铺后台点击【去使用】按钮，效果等同触发授权（无扩展字段state），授权成功后可根据软件架构形式、类目不同进行跳转处理；

自测地址：（授权链接，无state入参）

**https://fuwu.pinduoduo.com/service-market/auth?response_type=code&client_id={应用client_id}&redirect_uri={client_id对应的回调地址}**

**注意：服务发布成功前，暂无法从服务市场购买后【去使用】，需通过以上授权链接模拟测试该场景。**

![图片](https://t16img.yangkeduo.com/eden-admin/2021-03-25/a4b28d79-babd-45c9-abc8-13c563e367cd.png)

场景二：软件内店铺绑定

软件内拼接授权链接（可带有扩展字段state），用户点击授权弹出授权页面，授权成功后根据获取到的店铺授权信息，可刷新显示店铺信息列表。

自测地址：（授权链接，可增加state入参）

**https://fuwu.pinduoduo.com/service-market/auth?response_type=code&client_id={应用client_id}&redirect_uri={client_id对应的回调地址}&****state={自定义参数}**

### **类目功能划分要求**

**请确保功能完善可用，否则不予通过审核**

| **类目** | **功能划分要求（包含但不限于）** |
| --- | --- |
| 搬家上货 | 商品链接/ID采集、上传设置（价格、必填属性、其他必要信息等）、上传商品、批量上传、采集历史 |
| 商品优化分析 | 水印标签模板、水印/logo编辑、详情页制作、活动主题图模板、商品素材编辑、（批量）商品编辑（上下架、预售、库存、标题、分类、商品编码、运费） |
| 打单 | 订单同步、打印快递单、打印发货单、手工订单/自由打印、发货、打印日志、发货地址管理、电子面单模板设置、发货单模板设置 |
| 订单处理 | 订单同步、发货（可选）、售后管理、销售统计、细分类型订单追踪管理 |
| 进销存 | 商品管理、订单管理（打单、发货、售后）、销售管理、库存管理、采购管理、店铺管理 |
| 企业ERP | 商品管理、订单管理、采购管理、库存管理、售后管理、财务管理、报表管理、客户管理、店铺管理 |
| 多多推广 | 搜索推广（计划新建、编辑、删除、数据报表）、场景推广（计划新建、编辑、删除、数据报表） |

### **禁止出现违规功能**

[《拼多多开放平台应用开发安全管理规范》](https://open.pinduoduo.com/application/document/browse?idStr=9BE60F0BC49913C0)

[《拼多多开放平台违规处理规则》](https://open.pinduoduo.com/application/document/browse?idStr=9B1B1B489FCE704C)

注意：

1. 软件内禁止出现任何形式导出拼多多订单数据到外部文件、采集拼多多商品信息、搬家拼多多商品等功能

2. 软件内禁止出现登录店铺后台、内嵌页面授权等任何需要店铺账密登录的功能

3. 软件内禁止出现爬取店铺后台未提供接口的数据（如：店铺后台数据中心模块所含数据）

### **订单相关类目-加密改造验收标准**

完整改造方案参考：[文档中心-基础文档-安全保障](https://open.pinduoduo.com/application/document/browse?idStr=7553D76C8D1713EF)**（数据加密说明、订单类/企业ERP接入方案、前端解密接入风控）**

**加密说明：**

1.应用已配置全量加密，即通过订单API/订单同步服务获取的收件人信息字段均已加密；

2.isv在数据库只允许存储密文数据，历史数据需调用开平加密接口进行转换后存储（使用[pdd.open.kms.encrypt.batch](https://open.pinduoduo.com/application/document/api?id=pdd.open.kms.encrypt.batch)批量加密）

**解密率控制说明：（企业ERP<=8%;打单、进销存、订单处理<=5%）需持续控制解密率，否则会触发平台处罚**

1.建议使用拼多多电子面单，已支持密文收件人入参[pdd.waybill.get](https://open.pinduoduo.com/application/document/api?id=pdd.waybill.get)取号拼多多电子面单，无需解密后再取号；

2.前端订单收件人信息展示、导出、发货单打印，需使用[pdd.open.decrypt.mask.batch](https://open.pinduoduo.com/application/document/api?id=pdd.open.decrypt.mask.batch)批量数据解密脱敏接口，密文入参接口返回打码数据，如手机号：18*******11,不算解密；

3.软件改址需通过[pdd.order.update.address](https://open.pinduoduo.com/application/document/api?id=pdd.order.update.address)回传给平台，实际是否修改成功，以接口返回结果为准；若接口报错或调用失败，不允许私下为商家修改地址；

4.订单合单/查询功能可使用密文检索串，无需解密。具体实现可参考各类目接入方案中《订单合单业务改造》模块说明；

5.禁止提供批量解密功能，商家点击前端的脱敏信息-获取完整收件人信息（**隐私号改造说明**），需要对接OPLB+前端解密风控接入（应用上线后，迁移至多多云后再做对接，[前端解密规范](https://open.pinduoduo.com/application/document/browse?idStr=C628565D8883F05F)）

6.禁止将订单解密后发消费者短信，短信需走拼多多短信通道，且短信手机号码直接传密文。[短信服务入驻指南](https://open.pinduoduo.com/application/document/browse?idStr=74D221BBF6D69EA5)

**隐私号改造说明：**

涉及调用解密接口（[pdd.open.decrypt.batch](https://open.pinduoduo.com/application/document/api?id=pdd.open.decrypt.batch)批量数据解密接口）、前端查看完整收件人信息（oplb）的isv需改造且完成验收，因涉及入云操作可在服务发布前完成改造并验收。

隐私号改造方案：[https://open.pinduoduo.com/application/document/doc?fullScreen=true&docReference=virtualnum](https://open.pinduoduo.com/application/document/doc?fullScreen=true&docReference=virtualnum)

注意：先提供clientid+mallid申请虚拟号灰度测试，解密才能获取到虚拟号。
