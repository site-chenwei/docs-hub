# 前端解密规范

更新时间：2021-05-20 11:24:45

### 

基本规范

应用前端提供给商家点击解密的功能，必须要严格按照如下规范要求来实现，其中主要注意点如下：

1、默认展示给商家的是打码之后的数据，打码可使用 [脱敏接口](https://open.pinduoduo.com/application/document/api?id=pdd.open.decrypt.mask.batch) 实现

2、终端获取明文的 http 服务接口路径必须按照指定的路径规范来实现

3、出入参遵循规范

4、解密是 SLB 网关来解密，应用接口直接返回密文即可，如果不是规范返回会解密失败

5、只允许单个字段解密

### 

**接口路径规范**

如上文所述，开发者需要自己实现前端查看解密的接口。前端获取明文的调用请求，请求的路径必须严格遵循 /pdd/control/decrypt/v1/xxx 的格式。共有 6 个加密数据可以被解密，每一个数据的调用路径如下：

• <your\_domain>/pdd/control/decrypt/v1/receiverPhone （获取收货人手机号）

• <your\_domain>/pdd/control/decrypt/v1/receiverName （获取收货人姓名） • <your\_domain>/pdd/control/decrypt/v1/receiverAddress （获取收货人地址）

• <your\_domain>/pdd/control/decrypt/v1/cardnum （获取收货人身份证号）

• <your\_domain>/pdd/control/decrypt/v1/cardname （获取收货人身份证姓名）

• <your\_domain>/pdd/control/decrypt/v1/address（获取收货人的详细地址）

注：<your\_domain> 指的是开发者云内应用的域名，比如开发者在多多云上对外提供服务的域名是 service.pavelzhang.top，则解密收件人手机号字段的解密请求就应该调用 service.pavelzhang.top/pdd/control/decrypt/v1/receiverPhone。

这里 /pdd/control/decrypt/v1/receiverPhone 必须直接跟在域名后面，不可以再有其他的子路径（比如 /pdd/control/decrypt/v1/receiverPhone.php 或者 pdd/control/decrypt/v1/receiverPhone/ 都是不可以的）。

### 

**应用响应返回密文**

被调用的程序接口要符合以下几点：

1.  **接收指定调用路径的请求**
    
2.  **获取密文数据后，不再通过解密接口解密**
    
3.  **遵循请求参数和返回参数的规范进行调用和返回**
