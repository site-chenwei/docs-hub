# 子账号管理

更新时间：2020-05-26 00:04:13

随着服务商的规模扩大，对于开放平台功能的使用，需开放给各个员工进行操作与管理。因此，拼多多开放平台上线了子账号管理，主账号可以创建多个子账号，并对不同子账号分配不同权限，极大的方便了主账号对员工的管理，提高了工作效率。

### 

**一、主账号授权**

操作入口：[个人中心-子账号管理](https://open.pinduoduo.com/application/sub-account)

#### 

**1 成员管理**

![](https://testimg.yangkeduo.com/perse-open-api/2020-05-24/2d451fb2-d2c5-47cc-a901-5f70e9ad32cf.png)

1.1 点击[添加成员](https://open.pinduoduo.com/application/sub-account/create)，填入员工的账户信息，其中：

-   登录账号：将作为子账号登录使用的信息，不可修改，请谨慎填入，子账号登录名：账号@开发者id
    
-   显示名称：将作为在沟通工具knock及客服平台的显示名称，请尽量填写易于识别的代号或姓名
    
-   登录密码：将作为子账号登录使用的密码
    
-   绑定手机号：将作为子账号登录沟通工具knock的信息，目前同一手机号不可重复注册
    
-   登录验证：开启后，子账号每次登录都需填写短信验证码，出于安全考虑，建议开启
    

![](https://testimg.yangkeduo.com/perse-open-api/2020-05-24/fbd80f49-fe20-4159-bc77-fdc4015d0bdd.png)

信息创建完成后，会自动跳转至成员的权限配置页，可进行对应权限的勾选，关于权限的解释说明，详见本文1.2

1.2 已创建的成员，可在列表中进行查看，点击【详情】可查看该子账号的详细信息，包含基本账户信息及权限信息。

同时，【列表页】及【详情页】均有编辑成员基本信息和编辑权限信息的入口，可进行信息修改。

1.3 若子账号忘记密码无法登录，暂无子账号自助找回密码的入口，需由主账号信息密码重置，入口：【成员详情页】-【重置密码】

1.4 若员工离职，为保证账户数据安全，建议及时删除该子账号，入口：【成员详情页】-【删除账号】

#### 

**2 权限管理**

权限管理分为两个维度：操作权限、数据权限。操作权限指的是子账号在页面中进行的行为，比如增删改查等；数据权限指的是操作的数据对象。当用户对某项业务同时拥有操作及数据权限时，才能进行完整的功能访问。

权限配置流程分为以下三步：

**2.1 操作权限包管理**

操作权限通过权限包进行批量管理，操作入口：[个人中心-权限管理-操作权限包管理](https://open.pinduoduo.com/application/permission?tab=ACTION_PKG)

1）创建操作包

一般推荐按员工职能进行操作权限分配，比如根据不同级别的客服岗位设置不同操作权限，为普通一线客服岗位设置仅订单查看功能的权限包，对客服主管岗位额外配置含订单及退款操作的权限包。

2）创建完成后，可在权限包详情页查看已配置的权限明细、变更历史以及已配置该权限包的成员列表。

3）若需要修改权限包信息，可在权限包详情页点击【编辑基本信息】后操作修改。

4）若权限包不再使用，需清除数据，可通过权限包列表页【删除】以及权限包详情页【删除操作权限包】进行操作，注意：为避免误删除有效数据，若权限包关联了成员，则无法删除，需在成员配置中关闭该权限包配置后，才可删除成功。

**2.2 数据权限包管理**

数据权限通过权限包进行批量管理，操作入口：[个人中心-权限管理-数据权限包管理](https://open.pinduoduo.com/application/permission?tab=DATA_PKG)

1）创建数据包

一般推荐按业务范围进行数据权限配置，比如ERP开发者内部包含多项业务，为便于管理，可分别设置订单相关、商品相关、财务相关的不同数据权限包。

注：

当前仅云工作台需要配置数据权限，其他业务菜单暂不做权限拆分控制；

由于PaaS及SaaS数据已通过[分组管理](https://open.pinduoduo.com/paas/tag)进行过数据打包，此处权限控制不到实例维度，而是通过云分组进行数据打包。如需对云工作台内数据进行权限配置，请先至[云工作台-分组管理](https://open.pinduoduo.com/paas/tag)页面进行实例分组。

2）创建完成后，可在权限包详情页查看已配置的权限明细、变更历史以及已配置该权限包的成员列表。

3）若需要修改权限包信息，可在权限包详情页点击【编辑基本信息】后操作修改。

4）若权限包不再使用，需清除数据，可通过权限包列表页【删除】以及权限包详情页【删除数据权限包】进行操作，注意：为避免误删除有效数据，若权限包关联了成员，则无法删除，需在成员配置中关闭该权限包配置后，才可删除成功。

**2.3 成员权限管理**

操作入口：

1.  [个人中心-成员管理](https://open.pinduoduo.com/application/sub-account)\-列表中【编辑权限信息】
    
2.  [个人中心-成员管理](https://open.pinduoduo.com/application/sub-account)\-成员详情-【编辑权限信息】
    
3.  [个人中心-成员管理](https://open.pinduoduo.com/application/sub-account)\-添加成员完成页-自动跳转权限配置页
    

1）进行成员的权限配置前，请先确定已维护好必要的操作权限包和数据权限包。

2）根据成员的职能及负责的业务范围，选择配置相应的操作权限包和数据权限包，编辑页中的权限明细中，可预览已配置的权限信息

3）点击提交前，系统会判断是否存在不完整的配置项，如存在操作或数据权限其中一个维度配置缺失，会有如下图提醒，建议用户按自身需求进行成员权限配置。

tips：对于云分组数据，系统会判断到分组中的实例维度，若存在云服务有操作权限，但未配置云服务的实例，也会有异常提醒。

#### 

**3 操作日志**

入口：[个人中心-成员操作日志](https://open.pinduoduo.com/application/sub-account/operate-log)，该页面中，可查看：

-   主账号对成员及权限的操作记录
    
-   所有子账号在开发平台中的操作
    

### 

**二、子账号使用**

#### 

**1 登录及使用开放平台**

1.1 登录账号及密码由主账号在创建子账号时赋予，其中登录账号可由主账号在成员详情页进行查询，登录密码可在成员详情页进行重置。

**注：子账号登录开放平台时，登录账号为账号@开发者id，不可直接用注册手机号作为用户名登录**

1.2 登录后，子账号仅可见有权限的业务菜单，未被赋予权限的内容则无法访问。

#### 

**2 knock使用**

所有开放平台主账号&子账号均可使用knock沟通工具触达拼多多小二，使用注册所用手机号即可登录使用，具体关于knock的使用说明，详见文档：[https://open.pinduoduo.com/application/document/browse?idStr=63FBE02AA025FE8E](https://open.pinduoduo.com/application/document/browse?idStr=63FBE02AA025FE8E)
