# 方舟

更新时间：2026-02-24 19:59:48

## 

一、定义

1.  方舟 API：由拼多多定义的标准化应用程序接口（API）。
    
2.  方舟网关：方舟 API 的统一访问入口，同时服务于外部 ISV 和内部服务的调用需求。它通过固定域名[](https://ark-api.pinduoduo.com/)对外提供服务，实现了调用入口的收敛管理与标准化管控。
    

## 

二、场景类型​

根据调用方与被调用方的主体差异，方舟 API 的调用场景可分为两大类，覆盖不同业务协作需求：​

**（一）三方互通场景：ISV 间协作**

-   **场景定义：**调用方与被调用方均为 ISV，通过方舟 API 实现跨 ISV 的服务协作。
    
-   **调用链路：**ISV（调用方） → 方舟网关 → ISV（被调用方，通过target\_client\_id指定）。
    
-   **适用场景：**ISV 间需共享能力或协同完成业务时，如 A ISV 调用 B ISV 的物流查询服务、C ISV 调用 D ISV 的同步订单数据等。
    

**（二）官方集成场景：拼多多与ISV对接**

-   **场景定义：**官方集成是拼多多与外部系统对接的专属方式，目前由平台主动发起对接，暂不对外开放申请。调用方或被调用方中至少一方为拼多多官方服务，实现平台与外部的业务打通。​
    
-   **核心调用链路：​**
    
    -   拼多多官方主动调用 ISV 服务：拼多多官方服务 → 方舟网关 → ISV（被调用方，通过target\_client\_id指定）​
        
    -   ISV 调用拼多多官方服务：ISV（调用方） → 方舟网关 → 拼多多官方服务
        

## 

三、特色：多方实现与智能路由

方舟网关的核心价值体现在 “接口统一化” 与 “实现多样化” 的协同，具体通过以下两点实现：

**（一）同一API，多提供方支持**

单个方舟 API（如 “订单发货 API”“商品查询 API”）可同时接入多个 ISV 的实现方案，打破单一实现的限制，形成多元化的服务供给生态。

**（二）基于target\_client\_id的智能路由**

调用方在发起请求时，只需在请求参数中指定目标target\_client\_id，方舟网关即可自动识别并将请求精准路由至对应的 ISV 实现方。该机制既保障了接口定义的标准化，又满足了业务对实现方案多样化的需求。

## 

四、调用链路

方舟体系的完整调用流程可概括为三步，确保请求高效、精准流转：

1.  调用方构造请求，明确指定目标 API 及对应target\_client\_id。
    
2.  请求通过唯一入口[](https://ark-api.pinduoduo.com/)发送至方舟网关。
    
3.  方舟网关解析target\_client\_id，将请求路由至对应的 ISV 实现节点，完成服务调用并返回结果。
