# 三方互通场景接入方案

更新时间：2025-10-19 23:27:51

## 

一、接口实现方接入方案

### 

场景接入申请

-   如果场景未接入，进入 [开放管理 - 三方互通 - 标准场景](https://open.pinduoduo.com/application/ark/third-party?tab=gfcj) 页面，根据场景名和场景描述 选择需要接入的场景，点击 “接入” 按钮，选择需接入的应用，确认提交后完成申请。
    

![](https://t16img.yangkeduo.com/perse-open-api/2025-10-19/20454e39-fc7b-46ca-ab8d-1f78b6fa131e.png)

### 

接口开发

-   [开放管理 - 三方互通 - 已接入场景](https://open.pinduoduo.com/application/ark/third-party?tab=yjrcj) 页面点击对应的 “接口管理”，选择 “查看需实现 api”；查看接口文档，依据文档 入参规范 与 出参规范 完成后端接口开发（作为接口实现方，需确保接口功能符合平台规范）。
    

![](https://t16img.yangkeduo.com/perse-open-api/2025-10-19/df8e85dc-4d86-4477-b298-0e0b764999db.png)

**注意事项：**后端设计需要完全参照 **出参规范** 的参数进行命名。如字段名称对应不上，则不会被方舟网关透传出去。例如**出参规范**中的参数是 is\_success，但是后端返回的结果中的字段为 isSuccess，那么在方舟网关返回的response中将不会有 is\_success 和 isSuccess字段。

### 

接口自测验证

-   接口开发完成后，返回场景接入页面，点击 “接入”进入自测页面；
    

![](https://t16img.yangkeduo.com/perse-open-api/2025-10-19/d4d0e1a0-4bee-4e7c-8875-aa7a8696583f.png)

-   填写自身后端接口地址，发起自测请求，验证接口是否能正常接收并响应。
    

### 

配置发布

-   自测通过后，点击 “下一步”，填写已通过测试的后端地址，提交 “发布” 申请，发布成功后则全部接入完成。
    

### 

授权给接口调用方应用

-   “已接入场景” 页面，点击 “调用管理”；
    

-   点击 “新增授权”，输入需授权调用的调用方的client\_id（client\_id需向调用方获取），完成授权配置。
    

## 

二、接口调用方接入方案

### 

场景接入申请

-   进入 [开放管理 - 三方互通 - 标准场景](https://open.pinduoduo.com/application/ark/third-party?tab=gfcj) 页面，根据场景名和场景描述 选择需要接入的场景，点击 “接入” 按钮，选择需接入的应用，确认提交后完成申请。
    

### 

授权给接口实现方应用

-   点击 “调用管理”；
    

-   点击 “新增授权”，输入需授权调用的实现方的client\_id（client\_id需向实现方获取），完成授权配置。
    

### 

开发与联调

-   [开放管理 - 三方互通 - 已接入场景](https://open.pinduoduo.com/application/ark/third-party?tab=yjrcj) 页面点击对应的 “接口管理”，选择 “查看可调用 api”；查看接口文档，作为调用方，依据文档 **入参规范** 与 **出参规范** 进行开发。
    

-   开发完成后，点击 “联调”，在target\_client\_id字段填写接口实现方的client\_id，并输入接口入参发起联调；
    

-   联调验证：平台网关将请求转发至接口实现方配置的后端地址，调用方需与对接的接口实现方确认两项内容——①请求是否准确送达对方后端；②是否能正常接收对方返回的响应参数。两项均正常则联调通过。
