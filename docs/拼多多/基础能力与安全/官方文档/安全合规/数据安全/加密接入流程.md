# 加密接入流程

### 目标用户

已入云但未开启加密的开发者。

新入云的开发者已经默认开通了全量加密，可跳过此流程。

### 前置条件

开启加密的前置条件有以下几个

1、开通多多云云工作台

申请地址：https://open.pinduoduo.com/paas/self-apply

2、有一个测试店铺，平台不提供测试店铺

店铺注册地址 ims.pinduoduo.com/plus

### 整体流程

![图片](https://t16img.yangkeduo.com/perse-open-api/2020-09-30/38af03cf-5802-4a5d-83e0-e46c8c7ae2f2.png)

（1）进入开放平台需要进行加密的应用详情中，点击“数据加密”

![图片](https://t16img.yangkeduo.com/perse-open-api/2020-11-11/2d58cc9f-9a64-4613-99a5-92f76c3709cb.png)

（2）查看加密改造白皮书，阅读完之后点击“下一步”

![图片](https://t16img.yangkeduo.com/perse-open-api/2020-11-11/0f68320f-a6c0-4238-9c49-00aa78b73ecb.png)

（3）把需要测试的店铺id配置到名单中，点击“配置完成”，3分钟之后该店铺订单接   口以及dts增量数据的敏感字段将会以密文的形式返回

![图片](https://t16img.yangkeduo.com/perse-open-api/2020-11-11/4037083a-75f1-45c4-9e7b-e6b31f7606dc.png)

（4）若需要修改加密名单请点击“修改原批次”可对加密的名单进行修改，如果确认已加密的店铺不用回滚到明文状态可以点击“继续添加名单”，则前一批次的店铺将会永久加密（用于灰度切换加密）

![图片](https://t16img.yangkeduo.com/perse-open-api/2020-11-11/b43750fd-50ff-4683-aeb8-6bad9e779f0e.png)

### 小流量加密

（1）首先对单个店铺进行迁移测试

（2）按自己的需求对少量用户进行加密切换，

如果发现存在问题，需取消加密mall_id加密名单，可自行修改。

（3）监控一段时间之后如果没有问题可灰度剩下未加密的店铺id，请点击“继续添加名单”；也可对全量用户切换加密，请点击“全部数据加密”，注意：该操作不可逆，请谨慎操作。

### 全量加密

- 确认已经加密改造完成之后将应用切换到全部数据加密状态

- 在线上加密覆盖到全量用户后，接下去的一个重要步骤就是要把历史数据全量加密。

- 调用批量加密接口api: pdd.open.kms.encrypt.batch，对数据库中平台要求加密的数据进行数据更新

- 如果个别几个mall_id导致无法全量开启，可以设置加密mall_id不加密名单，请联系对接的运营人员。
