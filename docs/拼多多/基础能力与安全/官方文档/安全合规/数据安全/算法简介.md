# 算法简介

### 加密算法特点

- 加密算法使用“AES/CBC/PKCS5Padding”

- 加密用的初始向量会直接编码到加密数据中, 因此相同数据内容多次加密的结果不同

- 不同应用使用不同的秘钥，不同应用间加密数据无法解密

- 支持秘钥轮换, 密文中会包含加密密文用的秘钥版本，更新后支持老版本密文解密

- 密文可以反算出订单号、店铺等信息，如果发现泄漏可追踪

### 加密字段格式

加密后的完全密文结构如下：

文本字符串类（姓名、地址）构造：

| SEP | EncryptedData | SEP | Index | SEP | Version | SEP | SEP |
| --- | --- | --- | --- | --- | --- | --- | --- |

手机号和身份证类构造：

| SEP | Index | SEP | EncryptedData | SEP | Version | SEP | SEP |
| --- | --- | --- | --- | --- | --- | --- | --- |

其中 Index 是检索信息，同一份数据多次加密后的结果都不会变。

### 加密数据样例

加密算法支持模糊搜索(可选)，模糊加密的数据比一般加密更长一些，大概长度扩大 10-15 倍。

加密数据的示例如下：

| phone(手机号)加密方式，加密类型：phone |  |
| --- | --- |
| 手机号 | 18967038927 |
| 支持搜索密文 | $7AnwZJ1e6BZc$AAAAAADkt0hgZkt6KLklIDxVp+F1wzHsRsPUw0s19fk=$2$$ |
| 后八位手机号 | 67038927 |
| 67038927查询摘要 | $ 7AnwZJ1e6BZc$ |

| 身份证加密方式示例，加密类型：id |  |
| --- | --- |
| 身份证号 | 362102198902031310 |
| 支持搜索密文 | #xjH3#AAAAAADkt0ijWkYH83V7d9iZw0RbdzCXeKcfV3l2lGJV3OiASD6fpWMsB1ZsEEQc#2## |
| 查询串明文 | 1310 |
| 查询串摘要 | #xjH3# |

| 普通文本加密方式示例(地址, 昵称等) ，加密类型：simple |  |
| --- | --- |
| 文本 | 上海市长宁区X小区999栋999户 |
| 支持搜索密文 | ~AAAAAADkt0io3kp+aGA1Gq6S1D66O5bhi3ysXosALbSWXV6K+ BYciGWzfL7LZk/BDwebszQR5eDWAKLFYoSxgw== ~Ew/jZ2uN9cqK5wk1O4mgVINIu9B7nvYINXiZMm/QCuK1zALvOxCf5NMykFJ2~2~~ |
| 包含查询串明文 | 长宁区 |
| 查询串摘要 | 5wk1O4mg |

注 1: phone，id为有规律的数字, 其加密算法只支持使用尾号后8位搜索。如18967038927的手机号加密后可用67038927搜索.

注 2：因为搜索是摘要检索，搜索可能返回有冲突摘要的结果, 例如: 使用”长宁”搜索出来的结果最终还需要用户点击获取明文确认。

注 3：搜索的输入字符需要至少4个数字或者2个汉字. 如果数据本身长度不够4个数字或2个汉字, 则此时输入全文即可搜索.  如: 原始数据为"你"的, 使用"你"调用平台搜索接口即可获取搜索用文本.

注 4：加密之后的输出均为字符串, 如果原始字段为number类型. 平台会转成string然后加密. 此时api接口会在新字段上添加一个字段, 名为: encrypt_{原字段名}

### 密文长度

加密后由于增加了密文信息, 将比原文要长, 各种常见字段加密后的数据长度参考表如下:

| 数据类型 | 密文长度（支持搜索时） | 密文长度（不支持搜索时） |
| --- | --- | --- |
| 手机号（11 位） | 54 | 52 |
| 身份证号（18 位） | 74 | 53 |
| 一般数据(5字符,假设全中文) | 66 | 48 |
| 一般数据(10字符,假设全中文) | 106 | 68 |
| 一般数据(20字符,假设全中文) | 190 | 112 |
| 一般数据(40字符,假设全中文) | 354 | 196 |

注: 当发生加密秘钥version字段变更时, 密文长度可能在发生变化. 建议设计字段时在表中字段上加10个左右字符.
