# 2.X.X与5.X.X版本升级

如需将ohpm-repo版本2.X.X/5.X.X版本升级到更高版本，可参考此文档。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/b0/v3/AQu-PS06Qx-bHC1HCSp7WQ/notice_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=14A1391E9A4C5370468C4952C0B83977792E235B8D06BC7D6848C266ED83E93B)

1\. 在升级之前，请务必备份好ohpm-repo私仓工具中的历史数据，避免因升级操作失误，导致数据丢失。备份的内容包括[<deploy\_root>](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-upgrade-110_to_2xx#li10435216234)ohpm-repo部署根目录内数据，db元数据以及store三方包数据，详细可参考[数据备份](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-data-backup) 。

2\. 如果您想要改变db元数据和store三方包的存储方式，可在正确升级后参考[数据迁移文档指导](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-data-migration)修改。

1.  旧版本服务停止：如果旧版本的服务还在运行，升级版本前请停止，执行stop。
    
    ohpm-repo stop
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/7e/v3/oPUtkEfTSvCWAThHeqhrqg/caution_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=1515A63CF12E48EFFD7E500831A5B5DDA2230A25F0A6E9BCE59360B2067DF5E6)
    
    -   如果部署的是多实例，升级前需要停下所有机器中的ohpm-repo服务，再进行升级操作。
    -   若想在其他目录使用ohpm-repo，请将对应版本ohpm-repo工具包解压目录中bin目录的路径配置到[系统环境变量](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/ide-ohpm-repo-faq#section24117279211)path中。
    
2.  下载并解压工具包：下载新版本的ohpm-repo私仓工具包，并解压（请解压到一个空文件夹中）。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/9a/v3/dYwC0J2sToKRk6_JCyM0Zg/zh-cn_image_0000002530911554.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=72C3F48BBDB037460B4DC30CBA024006FF957BFBE7A8F39E75EE90CAAE8A696E)
    
3.  安装完成之后，进入ohpm-repo私仓工具包解压目录下的bin目录，执行如下命令：
    
    ohpm-repo -v
    
    终端输出为新版本的版本号，则表示解压成功。
    
4.  移植配置文件信息：ohpm-repo的版本更新前后都保持在2.X.X/5.X.X范围内，新版本配置文件可以直接复用旧版本配置文件。
    
    -   找到ohpm-repo旧版本的配置文件：如果旧版本未指定配置文件，使用默认配置文件：旧版本ohpm-repo解压根目录/conf/config.yaml。
    -   拷贝旧版本配置文件至新版本：复制旧版本的配置文件到新版ohpm-repo解压目录下的conf文件夹，覆盖新版本中的默认配置文件。
    
5.  （可选项）新版本如果需要使用新的部署目录<new\_deploy\_root>，需要手动迁移数据。
    
    -   升级ohpm-repo：按照步骤1-4，解压ohpm-repo并拷贝替换配置文件信息。
    -   建立新部署目录：判断指定的新部署目录<new\_deploy\_root>是否存在，不存在则新建，新部署目录需存在且为空。
    -   拷贝数据文件：拷贝旧版本部署目录<deploy\_root>下的全部文件至新部署目录中。
    -   修改新版本ohpm-repo配置文件：打开新版本ohpm-repo的解压根目录，进入conf目录下，修改配置文件config.yaml中配置项deploy\_root为新的部署目录<new\_deploy\_root>。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/d2/v3/8cRA9NeMTReiE1YVXxjKxg/caution_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=F31DA39EEB5EC51E1B1E1034FAF54219DFFF3276956CDFF9CE2DDD41472E7149)
    
    在使用新部署目录时，旧版本的部署目录中meta文件必须要迁移到新版本部署目录中，否则将导致使用meta加密组件加密的数据无法被正确解密。
    

6.  新版本服务启动：正确拷贝替换配置文件信息后，进入ohpm-repo私仓工具包解压目录下的bin目录，执行以下命令启动新版本ohpm-repo服务。
    
    -   执行安装命令：
        
        _ohpm-repo install_
        
        结果示例：
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/88/v3/YjjT-U-MRw63YJ4nW-sqsw/zh-cn_image_0000002561831481.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=E56AB458369EDB0DDA8AA70F771F9FDDF3ADCF51A0C3006140DAC4FF4F108975 "点击放大")
        
    -   刷新环境变量：安装成功后，必须根据给出的提示信息刷新环境变量，Windows系统和Linux/Mac 系统，有不同处理方式：
        -   Windows系统： 关闭当前窗口，重新开启一个窗口。
            
        -   Linux/Mac系统：
            
            -   5.0.1之前版本：在命令行中执行刷新命令：source ~/.bashrc或者.~/.bashrc ；
                
            -   5.0.1及以后版本：
                
                在命令行中执行刷新命令：当shell为bash时，执行_source ~/.bashrc_或者. _~/.bashrc_ ；当shell为zsh时，执行_source ~/.zshrc_或者._~/.zshrc_ 。
                
    -   执行start命令：
        
        _ohpm-repo start_
        
        结果示例：
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/29/v3/AhLxzzfzSvabO-y6QFSVfQ/zh-cn_image_0000002530751558.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=DC8434B04020DA52B9F8F362FC2F8211BBFAC63F9F679EA8120A80608D7A7768 "点击放大")
        
        ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/2b/v3/qX9s85EDRCy8Moc0hQoXDA/note_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013643Z&HW-CC-Expire=86400&HW-CC-Sign=7A88C0900E917A6BAE5991A652B583FB24087623DDADAE454291F774ECD27097)
        
        版本升级之前，如果浏览器中已访问ohpm-repo页面，版本升级之后请刷新ohpm-repo页面。
        
    
7.  多实例部署机器快速升级：在多实例部署中，其他机器同步骤1-6进行快速升级操作。
