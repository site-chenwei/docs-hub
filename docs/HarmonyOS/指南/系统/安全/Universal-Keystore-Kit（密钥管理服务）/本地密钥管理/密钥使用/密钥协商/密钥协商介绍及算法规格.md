# 密钥协商介绍及算法规格

为了避免恶意第三方获取到保密信息，密钥本身是不可以直接在设备间传输的。一般情况下，会采取密钥协商的方式，安全地在双方（或多方）间共享密钥。密钥协商只会传输密钥的公钥部分，私钥仍存留在设备内，以保护数据的安全性和机密性。

两台设备间进行密钥协商，双方各准备一个非对称密钥，并互换非对称密钥的公钥，以对端公钥与己端私钥协商出一个密钥，该密钥在两台设备相同。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/17/v3/d73NS8FETaig9yb9FGzd_w/note_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013553Z&HW-CC-Expire=86400&HW-CC-Sign=84CAAA84316A33788119AA86211E3C5B8D16B8B95C9D32E0CD34597D56A8C88A)

-   轻量级智能穿戴不支持密钥协商功能。
-   使用现有密钥别名作为协商结果密钥别名会把现有密钥覆盖。

从API 23开始支持[群组密钥](https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/huks-group-key-overview)特性。
