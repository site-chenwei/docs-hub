# 等待调试

开发者可以通过将某个应用设置为“等待调试模式”，然后当开发者需要对应用进行调试时，拉起应用即可快速进入调试。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/0d/v3/U1BJOyxZSCSxb5DsbHBj9Q/note_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=FB6FFD04B58D6133FE27F80853E753AE66ABC79763C8D1DB91A86E1A2AC7672E)

-   应用设置为“等待调试模式”后，此时如果启动普通的debug调试，将会取消当前的等待调试模式。
-   设置“等待调试模式”前，需要将应用安装到设备上。

#### 操作步骤

1.  在设备选择框中选择调试的设备，并单击Run > Attach to Process by Name。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/21/v3/0_ttD1EZRc-j-yzsIOck8g/zh-cn_image_0000002561753735.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=0409F0E141E29A05222A4E90FA3E39964001B295658A259989739C4BB346D967)
    
2.  选择需要设置为“等待调试模式”的应用（默认为当前工程），选择需要进行调试的调试类型。然后单击**Attach**，即可将该应用设置为“等待调试模式”。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/2c/v3/_K2mCfwqQGO8YTSGYsL4YQ/zh-cn_image_0000002561753733.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=9C78256B501DBB1EFFB5377526ECEF27D8635E91A18AEE3ACDA20FB234F82875)
    
    此时会在DevEco Studio底部显示一个等待进度条，在应用被拉起之前，将一直处于等待状态。可通过进度条右侧的取消按钮进行取消。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/8f/v3/XD0RKG-5TOCsu7PcLCTXEg/zh-cn_image_0000002530913786.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=25673FC4C0DBF7B884B7C37F2A705E48806F44368D5D0B1386B9AA7EBA2F3E9F)
    
3.  拉起设备端应用，此时将会进入调试。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/8f/v3/pu5f2FkFQTCOJQ6OOKwpAg/zh-cn_image_0000002530753796.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=E6DB63FB3A3502CDF8B8DC95F8AA3F7CD5EFB69C915DD6435E100EC0E4F7CFE6)
