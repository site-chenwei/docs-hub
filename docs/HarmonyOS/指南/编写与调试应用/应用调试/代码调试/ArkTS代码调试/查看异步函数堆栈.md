# 查看异步函数堆栈

从DevEco Studio 5.1.1 Beta1版本开始，开发者可通过打开异步堆栈跟踪开关、设置异步调用链深度来跟踪异步函数调用的顺序。

![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/9b/v3/EveagKuHSkSjQuKweDID9w/note_3.0-zh-cn.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=84874FC25895F3617FD9AD67A3CCE29EB2384BE6DAB87F2CC3C445290BD94AE0)

-   异步堆栈跟踪开关为全局设置，开启后所有工程都生效。
-   修改异步堆栈跟踪开关或异步调用链深度后，需要重新启动调试或启动新的调试会话才会生效。
-   setTimeout函数异步堆栈不生效。
-   异步堆栈不展示变量列表。

1.  点击**File > Settings**（macOS为**DevEco Studio > Preferences/Settings**） **> Build, Execution, Deployment > Debugger > Async Stack Traces**。
    
    -   勾选**Enable async stack traces**打开异步堆栈跟踪开关。
    -   设置异步调用链深度**Async call chain depth**大于0，才能在调试堆栈时展示调用链对应层数。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/4e/v3/ie43NqqeSMmaGV1I26Fxvg/zh-cn_image_0000002561753739.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=3A900C58F50C83AEB95BCEC6F3ED69964DDD1FF05BE61C71B4901FC87D8F8FB0)
    
2.  在异步调用链中设置断点，启动调试，命中断点后，堆栈列表将展示对应调用链层数。如果实际的调用链层数比设置的异步调用链深度小，则只展示实际调用链层数。每个异步调用链以**Async call from**分隔，后面是调用函数。
    
    ![](https://contentcenter-vali-drcn.dbankcdn.cn/pvt_2/DeveloperAlliance_scene_100_1/a7/v3/N8hbwAptQB2_0kPbTd2V1g/zh-cn_image_0000002561833719.png?HW-CC-KV=V1&HW-CC-Date=20260417T013650Z&HW-CC-Expire=86400&HW-CC-Sign=BB5DCEA8AF4AEF23E7CE9A935528E06C59F0510A4E3F2E5ACDCC9BFB5133FA28)
